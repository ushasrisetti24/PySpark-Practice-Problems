# PySpark-Practice-Problems

## 📖 About This Repository

This repository contains my practice solutions to PySpark ETL problems inspired by the [zillacode](https://github.com/davidzajac1/zillacode) repo.  
I am running these exercises locally by hosting Docker on my desktop, which allows me to simulate real‑world data engineering workflows in a containerized environment.

---

### 🎯 Purpose
- Strengthen my understanding of **PySpark transformations** 
- Practice **ETL techniques** for cleaning, anonymizing, and restructuring data.
- Gain hands‑on experience with **Dockerized Spark environments**, ensuring reproducibility and portability.
- Build a portfolio of practical solutions that demonstrate my ability to handle distributed data processing tasks.

---

### 🛠️ Technologies Used
- **Apache Spark (PySpark)**
- **Python 3**
- **Docker Desktop**

---

### 📂 What You’ll Find Here
- Jupyter notebooks with code for each question (`notebooks/`).
- Documentation of problem along with input/output DataFrame transformations.

---

### ✨ Portfolio Note
By combining PySpark with Docker, I’m creating a workflow that mirrors professional data engineering setups.  
Each script and notebook highlights a specific transformation challenge, along with examples of input and output DataFrames.
